# Standardize developer__handoff Tool Output Path to engineering_requests/

**Date:** 2026-05-12T10:35:51.028298
**Tenant ID:** 00000000-0000-0000-0000-000000000001

## Description
Currently the `developer__handoff` tool writes files to `docs/requests/`, while the canonical historical location for all engineering handoffs is `engineering_requests/`. This creates two competing paths that are out of sync, making it impossible to get a complete picture of pending, in-progress, and completed work without checking two locations.

The tool must be updated to write all new handoff files to `engineering_requests/` as the single source of truth. Any existing files in `docs/requests/` should be migrated to `engineering_requests/` and `docs/requests/` should be deprecated or removed.

## Acceptance Criteria
- [ ] All new files generated by developer__handoff are written to engineering_requests/ and never to docs/requests/
- [ ] Existing files in docs/requests/ are migrated to engineering_requests/ without data loss
- [ ] docs/requests/ is removed or deprecated after migration is confirmed
- [ ] If a file with the same name already exists in engineering_requests/, the tool appends a timestamp suffix rather than overwriting
- [ ] A README.md is added to engineering_requests/ documenting it as the canonical handoff location

## Technical Notes
Reference the existing engineering_requests/ folder structure and naming conventions already in use. The fix should be a config-level change to the tool's output path, not a code rewrite. Coordinate with the status prefix convention handoff (PENDING_standardize_handoff_status_prefix_convention) as both changes affect the same folder.
